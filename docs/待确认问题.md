# 待确认问题（问题+描述+建议）

| 问题 | 描述 | 建议 |
| --- | --- | --- |
| 非法字符规则边界 | 当前实现仅允许“中文/英文/数字/空格/中划线”，默认禁止括号、斜杠、点号等字符。PRD 只明确了部分非法字符示例，未完整枚举白名单。 | 建议确认是否允许 `()`、`/`、`.` 等常见业务字符；确认后我会同步调整正则并补回归用例。 |
| 分类名称唯一范围 | 当前按“全局生效分类名唯一”实现（`is_deleted=false`）。PRD 未明确是否按 `scope` 或 `parent_id` 分区唯一。 | 建议确认唯一范围是否应为：全局唯一 / 同 scope 唯一 / 同父节点唯一。 |
| 数据标准主键组合唯一策略 | 当前实现“生效数据中 `code+version` 唯一”。PRD 未明确软删除后是否允许重建相同 `code+version`。 | 建议确认软删除后是否允许复用相同版本号；如不允许，可改为全局唯一约束。 |
| 分类查询入参约束 | 当前按 PRD 执行“未传 `id` 时 `keyword` 必填”。这会限制“查询全部分类”场景。 | 建议确认是否需要开放“无 keyword 查询全部”能力（可增加 `allow_empty_keyword` 开关）。 |
| 关系查询方向 | 当前关系查询返回“出度 + 入度”关系。PRD 里有出入度描述，但历史接口可能仅期望 source 关系。 | 建议确认前端是否需要区分方向；如需要，我会增加 `direction=in/out/all` 参数。 |


|问题|处理意见|
| --- | --- |
| 非法字符规则边界 | 在当前允许范围内添加允许“括号、点号”字符|
| 分类名称唯一范围 | 确认唯一范围是否应为：同父节点唯一 |
| 数据标准主键组合唯一策略 | 软删除后允许复用相同版本号|
| 分类查询入参约束 | 开放“无 keyword 查询全部”能力，增加 `allow_empty_keyword` 开关 |
| 关系查询方向 | 保留当前现状，不做调整。关系查询相关的功能还没有设计完成|
